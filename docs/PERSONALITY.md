# 人格：個体化と遺伝（垂直継承）

> 実装: `emergence/personality.py` ／ 配線: `emergence/scenario.py`（`individuals=True` /
> CLI `--individuals`）。Issue #24。

## ねらい（ひと言）
住民を「出来合いの4タイプのコピー」から、**一人ひとり違い・親に似る**存在にする。
町の**文化図書館**が担う *水平（文化的）継承* に対して、これは **垂直（遺伝的）継承**で、
両者で「個」と「社会」が育つ町を成立させます。

## 3つの部品
1. **個体サンプリング** (`sample_individual`)
   各住民の7knobベクトルを、その町の**文化（プリセット）を中心**にしたガウス分布から引く。
   → 同じ「guardian の町」でも10体のクローンではなく、中心の周りに散らばった**個人の集団**。
   町ごとの*マクロな*差（穏やか／好戦的…）は legible なまま。
2. **本物の遺伝** (`blend`)
   子の成体ベクトル ＝ **両親の中点 ＋ 小さな突然変異**。
   旧実装の「親ラベルをコイン投げで丸ごと継承」を置き換え、世代を超えて
   **血筋・ドリフト・淘汰**が創発する。子は親の*文化ラベル*（key/label）は引き継ぎ、
   knob だけが混ざる。
3. **発達ウィンドウ** (`matured` ＋ `DevelopingBrain`)
   子の*実効*性格は、生まれた直後は**可塑的**（中立 0.5 から出発）で、幼少期
   （既定 `window_days≈2`）をかけて継承した成体ベクトルへ結晶化し、以後は**固定**。
   トレイトは**年齢のみの関数**で、状況では書き換えない——その場の状態（空腹・恐怖・
   承認欲求）は既に欲求レイヤーが*行動*として表現しており、**trait と state を混同しない**。

## 設計上の約束
- **人格はエンジンの外**（記憶と同じ原則）。`simulation.py` は住民の knob を一切知らない。
  配線は `scenario.py` と、出生時に呼ばれる newborn-brain ファクトリ（`TraitPool` を
  クロージャで閉じ込める）だけ。エンジンが持つのは粗い*文化ラベル*文字列のみ。
- **opt-in ＋ seeded**。フラグOFFでは全住民がプリセットそのもの＝
  `tests/test_baseline_contract.py`（4社会の決定論的結末）は**バイト等価で不変**。
  個体化の draw は seed 由来の専用RNGから取り、既定経路のRNG列を乱さない。
- 4プリセットは引き続き「**種となる文化（seed cultures）**」として名前付きで残る。

## 使い方
```bash
python3 -m emergence.cli --individuals --reproduction --days 20   # 個体化＋生殖→遺伝
```
コードからは `make_simulation(persona, individuals=True)`。
